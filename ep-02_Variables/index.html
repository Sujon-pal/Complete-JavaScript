üìö JavaScript Variables ‚Äî Explained in Depth

Variables store data so we can use and change it later in our code.

Modern JavaScript has three ways to create variables: var, let, and const. They look similar but behave differently.

---

‚úÖ 1. var (old way)

  • Introduced in early JavaScript (before ES6)
  • Function-scoped: visible inside the whole function
  • Hoisted: moved to the top of the function, starts as undefined
  • Can be updated and re-declared in the same scope
function demoVar() {
  console.log(a); // undefined
  var a = 10;
  console.log(a); // 10
}

var name = "Ali";
name = "Karim"; // ok
var name = "John"; // ok

‚ö† Not block-scoped: ignores { }, which can cause bugs. 

if (true) {
  var x = 5;
}
console.log(x); // 5 (still accessible outside the block)

‚ö†Ô∏è Not used much today because it can cause bugs.

---

‚úèÔ∏è 2. let (modern, ES6)

  • Introduced in ES6 (2015)
  • Block-scoped: works only inside { }
  • Can be updated but cannot be re-declared in the same scope
  • Hoisted, but stays in Temporal Dead Zone (TDZ) until declared ‚Üí using before declaration gives ReferenceError 
let age = 25;
age = 30; // ‚úÖ ok
let age = 40; // ‚ùå Error: already declared

if (true) {
  let y = 50;
}
// console.log(y); // ‚ùå Error: y is not defined
---

üîí 3. const (constant, ES6)

  • Block-scoped: same as let
  • Must be initialized when declared
  • Cannot be updated or re-declared
const pi = 3.14;
pi = 3.1416; // ‚ùå Error

‚ö† For arrays and objects: you can't reassign the variable, but you can change what's inside: 

const arr = [1, 2];
arr.push(3); // ‚úÖ ok
arr = [4, 5]; // ‚ùå Error
---

üß† Hoisting ‚Äî what it really means

  • var: hoisted and initialized as undefined
  • let and const: hoisted but not initialized ‚Üí ReferenceError if used before declaration (because of TDZ) 
console.log(a); // undefined
var a = 10;

console.log(b); // ‚ùå ReferenceError
let b = 20;
---

üì¶ Scope

  • Function scope: variable exists only inside a function ‚Üí used by var
  • Block scope: variable exists only inside { } ‚Üí used by let and const 
function test() {
  var x = 5;
}
// console.log(x); // ‚ùå Error: x not defined

if (true) {
  let y = 10;
}
// console.log(y); // ‚ùå Error: y not defined
---

---

‚ú® Best practice (modern JavaScript)

  • ‚úÖ Use const by default
  • ‚úÖ Use let if you know you'll change the value
  • ‚ö† Avoid var unless working with legacy code
