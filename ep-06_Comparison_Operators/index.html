üßÆ Comparison Operators in JavaScript (In-Depth)

1. == (Equal To)

Compares values after doing type conversion (loose equality).

let a = 5; let b = "5"; console.log(a == b); // true (because "5" is converted to 5)

‚ö†Ô∏è Use with caution due to implicit type coercion.

2. === (Strict Equal To)

Compares value and type. No type conversion.

let a = 5; let b = "5"; console.log(a === b); // false (number !== string)

‚úÖ Safer and recommended over ==.

3. != (Not Equal To)

Checks if values are not equal after type conversion.

let a = 10; let b = "10"; console.log(a != b); // false (they're equal after conversion)

4. !== (Strict Not Equal To)

Checks if values are not equal or types are different.

let a = 10; let b = "10"; console.log(a !== b); // true (different types)

5. > (Greater Than)

Returns true if the left value is greater.

console.log(7 > 5); // true console.log(3 > 8); // false

6. < (Less Than)

Returns true if the left value is smaller.

console.log(4 < 10); // true console.log(15 < 5); // false

7. >= (Greater Than or Equal To)

True if the left value is greater or equal.

console.log(10 >= 10); // true console.log(12 >= 9); // true

8. <= (Less Than or Equal To)

True if the left value is smaller or equal.

console.log(5 <= 5); // true console.log(3 <= 7); // true

üí° Bonus Tip: Comparing Different Types

When comparing numbers and strings, JavaScript tries to convert types automatically (unless you use ===).

console.log("100" > 20); // true ‚Üí "100" converted to 100 console.log("abc" > 5); // false ‚Üí "abc" can't be converted to number ‚Üí NaN
