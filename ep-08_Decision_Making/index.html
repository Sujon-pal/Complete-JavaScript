üîπ What is Decision Making?

In JavaScript, if, else if, and else are used to control the flow of your program based on conditions.

‚úÖ Basic If-Else Syntax

if (condition) {
  // code if true
} else if (anotherCondition) {
  // code if second condition is true
} else {
  // code if all above conditions fail
}

‚úÖ Example 1: Simple If-Else

Check whether a number is positive, negative, or zero.

<script>
  let number = -3;
  if (number > 0) {
    console.log("Positive");
  } else if (number < 0) {
    console.log("Negative");
  } else {
    console.log("Zero");
  }
</script>

‚úÖ Example 2: Grading System (Else If Chain)

<script>
  let score = 83;
  if (score >= 90) {
    console.log("Grade: A+");
  } else if (score >= 80) {
    console.log("Grade: A");
  } else if (score >= 70) {
    console.log("Grade: B");
  } else if (score >= 60) {
    console.log("Grade: C");
  } else {
    console.log("Fail");
  }
</script>

‚úÖ Why Use Else If and Else?

  • Prevents unnecessary checks
  • Improves performance
  • Makes code cleaner and easier to debug

‚úÖ Example 3: Login Validation (Practical Use)

<script>
  let userInput = "admin";
  let userPass = "1234";

  if (userInput === "" || userPass === "") {
    console.log("Username or password cannot be empty");
  } else if (userInput === "admin" && userPass === "1234") {
    console.log("Login successful!");
  } else {
    console.log("Login failed");
  }
</script>

‚úÖ Optimizing If-Else Conditions

  • Check the most likely or important condition first
  • Use else as a catch-all default
  • Avoid duplicate or overlapping conditions

‚úÖ Switch vs If-Else

If you're checking **fixed values**, use switch. If you're checking **ranges or complex conditions**, use if-else.

<script>
  let role = "admin";
  switch (role) {
    case "admin":
      console.log("Access granted");
      break;
    case "user":
      console.log("Limited access");
      break;
    default:
      console.log("No access");
  }
</script>

‚úÖ Truthy & Falsy in Conditions

Falsy values (treated as false in conditions):

  • 0
  • ""
  • null
  • undefined
  • NaN
<script>

  if (!value) {

‚úÖ Use of Quotes in JavaScript

You can use either 'single' or "double" quotes in strings. Consistency is important. Avoid mixing them.

let msg1 = 'Hello';
let msg2 = "World";

üß© Nested If-Else Statement in JavaScript

A nested if-else means an if or else block that contains another if-else inside it.

It is used when you need to make a **decision inside another decision**.

üîπ Syntax:

if (condition1) {
  if (condition2) {
    // code when both condition1 and condition2 are true
  } else {
    // code when condition1 is true, but condition2 is false
  }
} else {
  // code when condition1 is false
}

‚úÖ Example: Voter Eligibility Check (With Nested If)

<script>
  let age = 20;
  let hasVoterID = true;

  if (age >= 18) {
    if (hasVoterID) {
      console.log("‚úÖ You are eligible to vote.");
    } else {
      console.log("‚ùå You need a voter ID to vote.");
    }
  } else {
    console.log("‚ùå You are not eligible to vote.");
  }
</script>

‚úÖ Real-life Use Case: Student Exam Check

First check if student appeared in exam. Then, based on marks, assign grade.

<script>
  let attendedExam = true;
  let marks = 76;

  if (attendedExam) {
    if (marks >= 80) {
      console.log("Grade: A");
    } else if (marks >= 60) {
      console.log("Grade: B");
    } else {
      console.log("Grade: C");
    }
  } else {
    console.log("Result: Absent");
  }
</script>

‚úÖ Tips for Writing Nested If-Else:

  • Use nested if only when it logically makes sense
  • Keep levels of nesting minimal (2 or 3 max)
  • If nesting gets too deep, consider using functions or switch-case
