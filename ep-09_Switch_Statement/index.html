üìå What is a Switch Statement?

A switch statement is used when you want to perform different actions based on different values of a variable.

It's a cleaner way to replace multiple if-else-if statements.

üßæ Syntax

switch(expression) {
  case value1:
    // code block
    break;
  case value2:
    // code block
    break;
  default:
    // default code block
}

Note: break prevents the code from "falling through" to the next case.

‚úÖ Example 1: Simple Day Checker (Beginner)

<script>
  let day = 4;
  let dayName = "";

  switch(day) {
    case 1:
      dayName = "Sunday";
      break;
    case 2:
      dayName = "Monday";
      break;
    case 3:
      dayName = "Tuesday";
      break;
    case 4:
      dayName = "Wednesday";
      break;
    case 5:
      dayName = "Thursday";
      break;
    case 6:
      dayName = "Friday";
      break;
    case 7:
      dayName = "Saturday";
      break;
    default:
      dayName = "Invalid day";
  }

  console.log("Today is " + dayName);
</script>

‚úÖ Example 2: Calculator (Intermediate)

<script>
  let a = 10, b = 5, operator = "*";
  let result;

  switch (operator) {
    case "+":
      result = a + b;
      break;
    case "-":
      result = a - b;
      break;
    case "*":
      result = a * b;
      break;
    case "/":
      result = a / b;
      break;
    default:
      result = "Invalid operator";
  }

  console.log("Result: " + result);
</script>

‚úÖ Example 3: Grouping Multiple Cases (Advanced)

<script>
  let grade = "B+";

  switch (grade) {
    case "A+":
    case "A":
      console.log("Excellent");
      break;
    case "B+":
    case "B":
      console.log("Very Good");
      break;
    case "C":
      console.log("Passed");
      break;
    default:
      console.log("Invalid Grade");
  }
</script>

‚ö†Ô∏è Common Mistakes to Avoid

  • Forgetting break ‚Üí causes fall-through (executes next case too).
  • Using switch for ranges like x > 10 (use if-else instead).
  • Not including default (good to have for unknown cases).

ü§î When to Use?

  • Use switch when:
    • You check for exact values (e.g. "Admin", 1, "A+")
    • You want to simplify long if-else-if chains
  • Use if-else when:
    • You need to compare ranges or complex conditions

üõ† Debugging break in JavaScript

üìå What Does break Do?

  • Exits a switch, for, while, or do-while loop immediately.
  • Prevents executing the rest of the code inside that block.

‚ùå Common Bug: Forgetting break in switch

If you forget to use break, the code will "fall through" to the next case unintentionally.

<script>
let fruit = "apple";

switch (fruit) {
  case "apple":
    console.log("Apple selected");
  case "banana":
    console.log("Banana selected");
  default:
    console.log("Other fruit");
}
// Output:
// Apple selected
// Banana selected
// Other fruit
</script>

üõ† Fix: Add break after each case!

‚úÖ Correct Use of break in switch

<script>
let fruit = "apple";

switch (fruit) {
  case "apple":
    console.log("Apple selected");
    break;
  case "banana":
    console.log("Banana selected");
    break;
  default:
    console.log("Other fruit");
}
</script>

‚úÖ Use of break in Loop

Stop loop when condition is met.

<script>
for (let i = 1; i <= 10; i++) {
  if (i === 5) {
    break; // Exit loop when i is 5
  }
  console.log(i);
}
// Output: 1 2 3 4
</script>

‚ùå Bug: break Used Outside Loop/Switch

<script>
if (true) {
  break; // ‚ùå SyntaxError: Illegal break statement
}
</script>

‚úÖ Fix: Only use break inside switch or loops.

üîÑ break vs continue

  • break: Exits the loop/switch entirely.
  • continue: Skips current iteration and moves to the next one.
<script>
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;
  console.log(i);
}
// Output: 1 2 4 5
</script>

üß† Summary Tips

  • switch only checks for === (strict equality)
  • Always use break unless you want to fall-through
  • default works like else
  • Group multiple cases to reduce repetition
