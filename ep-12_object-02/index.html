üìò JavaScript Notes: Object.freeze() vs Object.seal()

üîπ Object.freeze()

  • Makes an object completely immutable.
  • Prevents:
    • Adding properties ‚ùå
    • Deleting properties ‚ùå
    • Modifying existing values ‚ùå
  • Object becomes non-extensible and read-only.

üîπ Object.seal()

  • Prevents adding and deleting properties ‚ùå.
  • Allows modifying existing values ‚úÖ.
  • Object becomes non-extensible but not immutable.

üß™ Examples

Object.freeze() Example


const user = {
  name: "Sujon",
  age: 25
};

Object.freeze(user);

user.age = 30;           // ‚ùå Doesn't change
user.city = "Dhaka";     // ‚ùå Won't add
delete user.name;        // ‚ùå Won't delete

console.log(user); // { name: "Sujon", age: 25 }

Object.seal() Example


const product = {
  name: "Laptop",
  price: 1000
};

Object.seal(product);

product.price = 1200;    // ‚úÖ Allowed
product.brand = "HP";    // ‚ùå Not allowed
delete product.name;     // ‚ùå Not allowed

console.log(product); // { name: "Laptop", price: 1200 }

üöÄ Advanced Topics

‚úÖ 1. Deep Freeze (Nested Freeze)

Object.freeze() only works on the top-level. For nested objects, we must recursively freeze:


function deepFreeze(obj) {
  Object.getOwnPropertyNames(obj).forEach(prop => {
    if (typeof obj[prop] === 'object' && obj[prop] !== null) {
      deepFreeze(obj[prop]);
    }
  });
  return Object.freeze(obj);
}

const settings = {
  theme: "dark",
  options: {
    fontSize: 14
  }
};

deepFreeze(settings);
settings.options.fontSize = 18; // ‚ùå Won't work

console.log(settings.options.fontSize); // 14
‚úÖ 2. Check if Object is Frozen or Sealed

const obj = { a: 1 };
Object.freeze(obj);

console.log(Object.isFrozen(obj)); // true
console.log(Object.isSealed(obj)); // true (freeze implies sealed)

üîç Real-World Use Cases

  • freeze():
    • Constants, Configs, Environment variables
    • Prevent accidental mutations in Redux/State
  • seal():
    • Data models where structure must remain fixed
    • Secure API response objects

‚ö†Ô∏è Limitations

  • Only shallow protection ‚Äî nested objects aren't affected (unless deep frozen).
  • Works only on objects, not primitives.
  • Doesn't prevent function execution inside objects (e.g., methods).

üìù Summary

  • Object.freeze() ‚Üí Lock everything (structure + values)
  • Object.seal() ‚Üí Lock structure, allow value changes
  • Object.preventExtensions() ‚Üí Block additions only
  • Use deep freeze for nested protection
  • Check immutability with Object.isFrozen() and Object.isSealed()
